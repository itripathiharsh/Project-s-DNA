# Glossary

## Purpose

This glossary defines key terms introduced across Phase 11 (Execution & Planning).

---

## E

### Epic

A collection of related features delivering a coherent capability. Each epic maps to one or more milestones and contains 5–15 tasks. Designed to be completed in 2–4 weeks. Defined in Phase 11/03.

### Execution Guide

The authoritative playbook for building Project DNA. Consolidates build order, task structure, validation gates, and quality standards. Defined in Phase 11/09.

---

## M

### Milestone

A concrete delivery gate with a definition of done, target date, and set of epics. V1 has 4 milestones (M1–M4), V2 has 3 (M5–M7), V3 has 3 (M8–M10). Defined in Phase 11/02.

---

## R

### Release Strategy

Versioning follows SemVer. Channels: nightly, alpha, beta, stable. Stable releases are quarterly. CI/CD automates publication to npm, Docker Hub, GitHub Releases. Defined in Phase 11/07.

### Risk Register

Living document tracking risks (LLM hallucination, performance on large repos, Ollama dependency, etc.) with likelihood, impact, and mitigation strategies. Defined in Phase 11/08.

### Roadmap

Three-version vision: V1 Local Cognition (individual dev), V2 Team Cognition (engineering team), V3 Organization Cognition (enterprise). Defined in Phase 11/01.

---

## T

### Task

Smallest unit of work — a single PR-sized chunk completed by one developer in 1–3 days. Each task has ID, title, effort estimate, dependencies, and acceptance criteria. Defined in Phase 11/04.
