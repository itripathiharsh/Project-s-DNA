## Purpose

Epics represent major bodies of work within each milestone. Each epic is a collection of related features that deliver a coherent capability to the user.

## Epic Template

Each epic should be documented with:

| Field | Description |
|---|---|
| **Goal** | What the epic delivers to the user |
| **Scope** | Components, interfaces, and files involved |
| **Dependencies** | Epics or external systems required |
| **Acceptance Criteria** | Measurable conditions for completion |
| **Tasks** | Individual work items (see Task Breakdown) |
| **Risk** | Technical or schedule risks |
