# Glossary

## Purpose

This glossary defines key terms introduced across Phase 12 (Future Vision).

---

## C

### Cross-Repository Cognition

The ability to analyze dependencies across repositories — detecting inter-service calls, resolving API contracts, and building a unified dependency graph. V3 capability. Defined in Phase 12/05.

---

## E

### Ecosystem Cognition

The ability to model the external software ecosystem — dependencies, CVEs, maintenance health, upgrade risk. V4 capability. Defined in Phase 12/06.

### Extensibility

The ability to extend DNA at every layer via plugins — custom engines, new data sources, third-party integrations. Plugin architecture uses stable ABI and sandboxed execution. Defined in Phase 12/08.

---

## F

### Future Vision

The long-term aspiration for DNA: a cognitive layer for software engineering spanning local, team, and organization cognition. Defined in Phase 12/01.

---

## O

### Open Problems

Unsolved challenges including the hallucination ceiling, repository scale, the novelty problem, the evaluator problem, and the privacy paradox. Defined in Phase 12/09.

### Organization Cognition

The V3 vision of DNA as a persistent, queryable representation of the entire engineering organization's technical knowledge — spanning repositories, people, and processes. Defined in Phase 12/04.

---

## R

### Research Directions

Open research questions: incremental cognition, predictive cognition, multi-language semantic models, natural language code search, and self-improving cognition. Defined in Phase 12/07.

---

## V

### V2 Roadmap

Team Cognition — multi-user, PostgreSQL, auth, webhooks, Slack bot, CI/CD integration. Defined in Phase 12/02.

### V3 Roadmap

Organization Cognition — plugin system, VSCode extension, SSO, audit logging, on-premise deployment. Defined in Phase 12/03.
